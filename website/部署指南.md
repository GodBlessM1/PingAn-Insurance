# 网站部署指南

## 方案1：使用Netlify（推荐，最简单）

### 步骤：
1. 访问 https://www.netlify.com/
2. 点击 "Sign up" 注册账号（可使用GitHub账号）
3. 登录后，将 `website/` 文件夹直接拖拽到 Netlify 网页
4. 等待上传完成（约1-2分钟）
5. 获得公网地址，例如：`https://your-site-name.netlify.app`

### 优点：
- 完全免费
- 自动HTTPS
- 全球CDN加速
- 支持自定义域名
- 部署简单，拖拽即可

## 方案2：使用GitHub Pages（免费）

### 步骤：
1. 访问 https://github.com/ 注册账号
2. 创建新仓库，命名为 `pingan-insurance-report`
3. 将 `website/` 目录内容上传到仓库
   - 方法A：使用GitHub网页界面上传
   - 方法B：使用Git命令行上传
4. 进入仓库设置（Settings）
5. 找到 "Pages" 选项
6. 在 "Source" 下选择 "Deploy from a branch"
7. 选择 `main` 分支，点击 "Save"
8. 等待部署完成（约1-2分钟）
9. 访问：`https://your-username.github.io/pingan-insurance-report`

### 优点：
- 完全免费
- GitHub托管，稳定可靠
- 支持自定义域名
- 版本控制

## 方案3：使用Vercel（免费）

### 步骤：
1. 访问 https://vercel.com/
2. 注册账号（可使用GitHub账号）
3. 点击 "New Project"
4. 导入GitHub仓库或直接上传
5. 点击 "Deploy"
6. 等待部署完成
7. 获得公网地址

### 优点：
- 完全免费
- 部署速度快
- 支持自动更新
- 全球CDN

## 方案4：使用云服务器

### 步骤（以阿里云为例）：
1. 购买云服务器（ECS）
   - 选择：1核2G，带宽1-2Mbps
   - 操作系统：CentOS 7或Ubuntu 20.04
   - 价格：约50-100元/月

2. 安装Nginx
   ```bash
   # CentOS
   yum install nginx -y
   systemctl start nginx
   systemctl enable nginx

   # Ubuntu
   apt install nginx -y
   systemctl start nginx
   systemctl enable nginx
   ```

3. 上传网站文件
   - 使用FTP工具（如FileZilla）
   - 或使用SCP命令
   - 将文件上传到 `/usr/share/nginx/html/`

4. 配置防火墙
   - 开放80端口（HTTP）
   - 开放443端口（HTTPS）

5. 访问服务器公网IP

### 优点：
- 完全控制
- 可部署动态网站
- 支持自定义配置
- 数据安全

### 缺点：
- 需要付费
- 需要技术维护

## 方案5：使用内网穿透（临时测试）

### 使用ngrok
1. 下载 ngrok：https://ngrok.com/
2. 注册账号
3. 启动本地服务器
   ```bash
   cd website
   python -m http.server 8080
   ```
4. 在另一个终端运行
   ```bash
   ngrok http 8080
   ```
5. 获得临时公网地址（有效期有限）

### 优点：
- 快速测试
- 无需购买服务器
- 适合演示

### 缺点：
- 需要本地电脑一直运行
- 地址不稳定
- 免费版有限制

## 自定义域名（可选）

### Netlify配置自定义域名：
1. 在Netlify项目设置中点击 "Domain management"
2. 点击 "Add custom domain"
3. 输入您的域名
4. 按提示配置DNS解析

### GitHub Pages配置自定义域名：
1. 在仓库设置中添加自定义域名
2. 配置DNS解析（CNAME记录）
3. 启用HTTPS（可选）

## 安全建议

1. **不要上传敏感数据**
   - 检查网站内容
   - 确保没有个人信息

2. **使用HTTPS**
   - 大部分托管服务自动提供HTTPS
   - 确保访问地址是 `https://`

3. **定期更新**
   - GitHub Pages自动更新
   - Netlify支持自动部署

4. **访问控制（如需要）**
   - 使用密码保护
   - 或限制访问IP

## 常见问题

**Q: 部署后图表无法显示？**
A: 确保图表文件路径正确，使用相对路径 `charts/xxx.png`

**Q: 访问速度慢？**
A: 使用CDN加速，大部分托管服务已提供

**Q: 想要修改内容？**
A: 修改本地文件后重新上传即可

**Q: 如何让更多人看到？**
A: 分享访问链接，或在社交媒体推广
